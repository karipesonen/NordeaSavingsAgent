Response 1 - 07-05-2026
Keywords: AI agents
Topic: Building a clear mental model and demo direction
Core Idea
An AI agent is not just “AI that talks.” It is a system that can:


Perceive information from the user, environment, files, sensors, apps, or tools.


Reason or plan what should happen next.


Act through tools, APIs, interfaces, or messages.


Observe feedback from the result.


Adapt its next step based on that feedback.


A useful simple loop is:
User goal   ↓Agent interprets goal   ↓Agent plans next step   ↓Agent uses tool / asks user / updates state   ↓Agent observes result   ↓Agent continues, stops, or hands control back
The key question is not “Does it use an LLM?”
The key question is:

What decisions does the system make, what actions can it take, and how does the user stay in control?


Where This Fits in AI Agents / Future Interfaces
Here is the clean distinction you should keep in your head:
System typeWhat it doesExampleChatbotResponds in conversation“Explain reinforcement learning.”Workflow automationRuns predefined steps“When I get an invoice, save it to Drive.”AI assistantHelps with tasks, often interactively“Summarize my emails and draft replies.”AI agentPursues a goal through reasoning, tool use, state, and feedback“Plan my trip, compare options, ask before booking.”Multi-agent systemMultiple specialized agents coordinate or debateResearch agent + planner agent + critic agentFuture interfaceNew mode of interacting with computationVoice assistant, AR overlay, robot, ambient assistant, wearable agent
A system becomes more agentic when it has more of these:
Goal understandingTool usePlanningMemory / stateAutonomyFeedback handlingRecovery from errorsUser control
But more autonomy is not automatically better.

Likely Weak Point
The most common beginner mistake is thinking:

“If my app uses ChatGPT, it is an agent.”

That is usually false.
A chatbot that answers questions is not necessarily agentic.
A good demo should show a loop, not just a response.
Weak demo:
User asks: “Help me study.”AI replies with study tips.
Stronger demo:
User uploads syllabus.Agent identifies upcoming deadlines.Agent proposes a study plan.User approves.Agent creates flashcards.Agent quizzes user.Agent notices weak topics.Agent adapts the next session.
The second system has perception, planning, action, feedback, and adaptation.

Correct Mental Model
Think of an agent as a delegation interface.
The user does not merely ask for information.
The user delegates part of a task.
But delegation creates design questions:
What may the agent do alone?What requires confirmation?What should it never do?How does the user inspect its reasoning?How can the user interrupt or correct it?How does it recover from mistakes?
A strong agentic interface is not one where the AI does everything.
It is one where the AI takes useful initiative while keeping the human meaningfully in control.

Design or Implementation Guidance
For a course demo, do not start with tools or APIs. Start with the interaction loop.
Use this template:
User goal:Agent observes:Agent decides:Agent acts:User reviews:Agent adapts:Success condition:Failure recovery:
Example:
User goal: Prepare for a difficult lecture.Agent observes: Syllabus, slides, user quiz answers.Agent decides: Which concepts the user is weak on.Agent acts: Generates targeted explanations and quiz questions.User reviews: Marks explanations as helpful/confusing.Agent adapts: Changes difficulty and focus.Success condition: User improves on weak-topic quiz.Failure recovery: Agent says uncertainty, asks user to choose topic.
That is already a convincing agentic demo.

Demo Project Implication
A strong demo should be small, visible, and interactive.
Bad project scope:

“An AI agent that manages my entire life.”

Better project scope:

“An AI study coach that reads one course topic, quizzes the student, detects weak concepts, and adapts the next explanation.”

Even better:

“An AI debugging tutor that watches a beginner’s code attempts, asks guiding questions, runs tests, and gives hints before revealing the fix.”

The best demos usually show:


A user has a real problem.


A normal chatbot would be insufficient.


The agent takes structured action.


The user can approve, correct, or interrupt.


The system handles at least one failure case.



Evaluation Criteria
When judging your idea, use these questions:
CriterionQuestionUsefulnessDoes this solve a real user problem?Agentic behaviorWhat decision is the agent actually making?Interface qualityWhat does the user see, approve, correct, or reject?FeasibilityCan you build a working version with limited time?SafetyWhat could go wrong if the agent acts confidently but incorrectly?TrustCan the user inspect or control important actions?Demo clarityCan the audience understand the loop in under 60 seconds?

Heres course description

About the Course

An AI agent is a computer program that can perform tasks autonomously on behalf of a user or another program. AI agents will be increasingly prevalent in the professional world, so it is important for students to know how to design, build and use AI agents that solve real-world problems. This is what you will learn in this course.

Designing AI agents requires combining knowledge about how AI agents work technically with knowledge about systems, organizations and human behavior. This is why this course has been designed with a broad range of students in mind to get a mix of perspectives on designing and interacting with AI agents. It is open to all Master and PhD level students from any disciplines. Students will work in multidisciplinary teams, ideally including technology, design and business perspectives. No deeper technical knowledge is required.

After the kick-off session, the course continues with a 24h hackathon, followed by four weeks of guided work. Students receive guidance from an instructor and present and demo their solution to an audience at a community event. Both the hackathon and community event include nice food and drinks for the participants.

Note that the course has mandatory attendance, including one weekend hackathon. The course is graded as pass/fail.
Learning objectives

This is a project course where the students learn the following:

    Critically reflect on the role AI agents play in professional contexts
    Understand technology, design and user perspectives on creating AI agents that interact with the real world
    Apply this knowledge to design and demo a concept for an AI agent to solve a problem, taking into account human and organizational realities
    Work in a multidisciplinary team

Response 2 - 07-05-2026
Keywords: Course strategy
Topic: Understanding what this AI agents course is really asking from you
Core Idea
This course is not mainly about building the most technically advanced AI agent.
It is about showing that you can design an AI agent for a real professional context while understanding:
Technical behavior+ Human behavior+ Organizational reality+ Interface design+ Responsible autonomy
So your project should not be judged only by “Does the prototype work?”
It should be judged by:

Did your team understand the real task, design a useful agentic interaction, build a convincing demo, and explain the tradeoffs clearly?

That is important because the course is open to non-technical students and graded pass/fail. The expected outcome is likely a concept + prototype + demo + reflection, not a production-grade agent.

Where This Fits in AI Agents / Future Interfaces
The course description gives a very practical definition:

An AI agent performs tasks autonomously on behalf of a user or another program.

But for your project, you should interpret “autonomously” carefully.
Autonomy does not mean:
The AI does everything without asking.
A better interpretation is:
The AI can take initiative inside a bounded task,use information/tools,make intermediate decisions,and return control to the human at the right moments.
For professional contexts, this is the key distinction:
Weak interpretationStrong interpretation“The agent answers questions.”“The agent helps complete a work process.”“The agent is autonomous.”“The agent has bounded autonomy with checkpoints.”“The demo uses ChatGPT.”“The demo shows an interaction loop.”“The AI is smart.”“The system is useful, controllable, and recoverable.”“We built features.”“We solved a workflow problem.”

Likely Weak Point
The hidden trap in this course is that teams may build a generic AI assistant.
Examples of weak concepts:
AI agent for bankingAI agent for sustainabilityAI agent for productivityAI agent for customer service
These sound relevant, but they are too broad.
A stronger project narrows the professional situation:
An AI agent that helps bank advisors prepare for client meetings by summarizing relevant client history, flagging missing information, and drafting a meeting agenda for human approval.
Now the agent has:
User: bank advisorProblem: preparation takes time and requires checking multiple sourcesAgent task: gather, summarize, flag, draftHuman control: advisor reviews before useFailure mode: missing or incorrect informationDemo: show before/after workflow
That is much more convincing.

Correct Mental Model
For this course, think of an AI agent as a workplace delegation system.
The important design question is:

What part of a professional workflow can safely be delegated to an AI agent, and what must remain under human judgment?

A professional AI agent usually sits between:
Human goal↓Organizational constraints↓Available information↓AI planning and tool use↓Human review↓Real-world action
Text diagram:
[Professional user]       ↓ gives goal[Agent interface]       ↓ interprets task[Planning / reasoning layer]       ↓ chooses steps[Tools / data sources / APIs]       ↓ returns results[Agent synthesizes output]       ↓ asks for approval or correction[Human decides final action]
Your demo should make that loop visible.

Design or Implementation Guidance
When you receive the company/instructor brief at the hackathon, do not immediately start building. First translate the brief into this structure:
1. Who is the user?2. What professional task are they trying to complete?3. What makes the task difficult today?4. What can the agent observe?5. What can the agent do?6. What decisions does the agent make?7. Where does the human approve, correct, or stop it?8. What failure would be dangerous, embarrassing, or costly?9. How will the demo show the agent loop clearly?
This is likely more important than choosing the tech stack on day one.

Demo Project Implication
Because the course has a 24h hackathon plus four weeks of guided work, the best strategy is:
Hackathon:Define the problem, user, agent loop, first prototype, and story.Four-week period:Refine the interface, add realistic data/tool behavior, test failure cases, polish the demo.Final event:Show a clear professional before/after scenario with a believable agent.
A strong demo should have one clear “wow” moment, but not too much scope.
Good demo pattern:
Before:A professional manually searches, compares, writes, coordinates, or checks something.During:The agent takes over part of the workflow.After:The human reviews an improved recommendation, draft, plan, alert, or decision support output.
The agent should not just produce text. It should appear to do work.
Examples:
Searches documentsCompares optionsFlags risksAsks clarifying questionsDrafts an action planUpdates a task boardGenerates a reportSimulates a conversationChecks policy constraintsAdapts after user feedback

Response 5 - 07-05-2026
Keywords: Wearable banking
Topic: Brainstorming smart-glasses banking agent uses across domains and scenarios

Core Idea

This project is much more commercially plausible than PALL0, but also more constrained.

The core question is:

What banking interaction becomes better when it is glanceable, hands-free, contextual, and gesture-controlled?

The project brief is about a Nordea banking agent running on a smart-glasses concept using a Dispelix waveguide display and Oura_DoublePoint gesture input. The brief explicitly asks the team to identify which banking moments genuinely benefit from glanceable + gestural interaction, and which ones do not belong on glasses. 

Nordea Wearable Banking Project

So the creative lens is:

Smart glasses = glanceable display
Gesture input = subtle control
Banking agent = financial interpretation + next action
Context = where/when money decisions happen
Trust layer = privacy, security, consent, confirmation

This is not “put the mobile banking app on glasses.”
That would be weak.

The stronger idea is:

A wearable banking agent helps you notice, decide, confirm, or avoid financial actions at the exact moment they matter.

1. Everyday Spending Awareness
Contextual Spending Whisperer

You walk into a café, grocery store, airport, or shopping mall. The agent gives a tiny glanceable financial cue.

“Monthly eating-out budget: 82% used.”
“Similar purchase last week: €14.20.”
“Cheaper subscription already covers this.”

Gesture controls:

Pinch = more detail
Double tap = dismiss
Hold gesture = save note
Swipe gesture = compare alternatives
Agentic behavior

The agent does not just show balance. It interprets context:

Detects location or merchant category
Checks spending pattern
Compares against budget
Decides whether to stay silent or intervene
Suggests a next action
Ridiculous version

A “financial conscience goblin.”

You reach for overpriced airport coffee.

Glasses: “This is your fourth €7 coffee this week. I am not angry, just disappointed.”

Absurd, but the real concept is useful: ambient spending awareness without opening an app.

2. Payment Confirmation and Fraud Prevention
Glanceable Transaction Guardian

When a transaction happens, the glasses show a minimal confirmation:

“Card payment: €42.80 at Prisma.”
“Recognized pattern: normal.”
“Gesture to approve if this was you.”

This is stronger for suspicious transactions:

“Unusual online purchase: €389.00.”
“Merchant: unknown.”
“Approve / freeze card / call bank?”
Agentic behavior

The agent evaluates transaction risk and proposes actions.

Low risk → silent log
Medium risk → glanceable confirmation
High risk → interrupt and request explicit approval
Ridiculous version

A “financial bodyguard.”

A suspicious transaction appears and the glasses go full spy movie:

“Threat detected. Freeze card?”

User makes a subtle pinch gesture. Card freezes.

Actually demo-worthy.

3. In-Store Purchase Decision Support
Should-I-Buy-It Agent

User looks at a product or scans a receipt/price tag with phone/glasses integration.

“You can afford this, but it delays your savings goal by 3 days.”
“Similar item bought 2 months ago.”
“This store has a 30-day return policy.”

This is not moralizing. It gives decision context.

Better than phone?

Yes, if the interaction is fast and private enough:

Look → glance → gesture → continue

No app opening. No standing awkwardly in the aisle.

Ridiculous version

A “luxury temptation detector.”

You look at designer shoes.

Agent: “I have prepared three arguments against this purchase and one emotionally supportive argument for it.”

Funny, but useful for impulse control.

4. Travel Banking Agent
Airport / Travel Money Companion

When traveling, banking friction increases: currency conversion, fees, card limits, scams, insurance, unfamiliar ATMs.

The glasses could show:

“Card fee here: 2.5%.”
“ATM withdrawal fee likely high.”
“Use credit card, not debit, for hotel deposit.”
“Travel insurance included with your card if purchase was made with it.”

Gesture input:

Pinch = show conversion
Tap = save receipt
Swipe = show safer alternative
Agentic behavior

The agent notices context and selects relevant financial guidance.

Location: airport / foreign country
Merchant: ATM / hotel / rental car
User account: card type / limits / insurance
Action: recommend best payment method
Ridiculous version

A “tourist scam radar.”

Walking near a currency exchange kiosk:

“Terrible exchange rate. Keep walking.”

This is actually quite compelling.

5. Personal Finance Coach
Micro-Coaching Agent

Instead of monthly reports no one reads, the wearable agent gives tiny financial nudges during daily life.

“You are on track this week.”
“Your transport spending is unusually high.”
“You have enough buffer to move €50 to savings.”

The key is restraint. It should not constantly nag.

Agentic behavior

It decides when not to speak.

If insight is useful now → show
If not urgent → save for weekly summary
If sensitive → require private context
Ridiculous version

A “financial Tamagotchi.”

Your savings goal is represented as a tiny creature in the corner of your display.

Save money → creature grows
Impulse spending → creature looks betrayed

Bad for serious banking branding, maybe good for youth financial literacy.

6. Accessibility and Inclusive Banking
Hands-Free Banking for Limited Mobility

Smart glasses and gestures could help users who struggle with phone interfaces.

Use cases:

Check balance
Confirm known payments
Hear/read simplified transaction summaries
Get scam warnings
Navigate banking support

This could be especially valuable for users with motor limitations, older adults, or people who find app navigation hard.

Agentic behavior

The agent adapts interaction complexity.

Simple mode
Large text
Voice + gesture confirmation
Step-by-step support
Human support handoff
Ridiculous version

A “banking co-pilot for people who hate banking apps.”

The agent says:

“I found the form you were looking for. Yes, they hid it under three menus again.”

Not ridiculous emotionally. Many people feel this.

7. Elder Fraud and Scam Protection
Real-Time Scam Shield

This is one of the strongest serious concepts.

Imagine an older user receives a suspicious call or message asking for money.

The glasses show:

“Warning: caller is creating urgency.”
“Bank staff will never ask for this code.”
“Gesture to call Nordea support.”
“Gesture to freeze outgoing transfers.”
Agentic behavior

The agent detects scam patterns and intervenes.

Observes message/call transcript
Recognizes manipulation pattern
Compares against bank safety rules
Recommends protective action
Escalates to trusted contact or bank support
Why glasses?

Because scam decisions happen under pressure. A phone notification may be ignored or be part of the scam itself. A glanceable overlay could interrupt at the right moment.

Ridiculous version

A “grandparent fraud force field.”

The agent appears like a tiny legal guardian:

“Do not send €2,000 to ‘your grandson’ until we verify that he exists.”

Comedic phrasing, serious use case.

8. Small Business Banking
Entrepreneur Cashflow Agent

For small business owners, financial decisions happen while working, not while sitting in spreadsheets.

Examples:

Restaurant owner sees daily cashflow summary during opening.
Freelancer gets tax reserve reminder after payment arrives.
Shop owner sees low liquidity warning before approving supplier order.

Gesture controls:

Tap = show cashflow
Pinch = approve draft invoice
Swipe = defer
Hold = call accountant
Agentic behavior

The agent monitors events and recommends next actions.

Invoice paid → suggest tax allocation
Expense arrives → categorize
Cash low → delay nonessential payment
VAT deadline near → prepare summary
Ridiculous version

A “tiny CFO in your glasses.”

At a café:

“Congratulations, you sold 94 cinnamon buns. Unfortunately, your margins remain emotionally damaging.”

Could be a strong demo if made tasteful.

9. Banking for Gig Workers
Income Volatility Agent

Gig workers and freelancers often have irregular income. The agent could provide real-time budgeting support.

“You received €180.”
“Recommended split: €45 tax, €25 savings, €110 usable.”
“Gesture to apply.”

It could also warn:

“Next rent payment in 6 days. Avoid discretionary spending over €30 today.”
Why wearable?

Because spending and earning decisions happen while moving through the day.

Ridiculous version

A “financial survival HUD.”

Like a video game overlay:

Rent shield: 72%
Tax danger: high
Coffee permission: denied

Absurd, but visually memorable.

10. Mortgage and Home Buying
Apartment Viewing Agent

While visiting an apartment, the glasses show financial implications:

“Estimated monthly payment: €1,420.”
“Maintenance fee higher than area average.”
“Commute cost impact: +€86/month.”
“Renovation risk: ask about plumbing.”

Gesture:

Pinch = save apartment
Swipe = compare to previous
Hold = generate questions for realtor
Agentic behavior

The agent combines bank data, user preferences, mortgage estimate, and viewing context.

Looks at apartment listing
Calculates affordability
Flags hidden monthly costs
Prepares follow-up questions
Updates shortlist
Ridiculous version

A “house-hunting truth overlay.”

You look at a beautiful apartment.

Agent: “Romantic balcony detected. Financial overextension also detected.”

This is one of the best demo scenarios because the value is easy to understand.

11. Student Banking
Student Money Survival Agent

For students, the agent could manage low-budget decisions.

“Lunch budget this week: €23 left.”
“Student discount nearby.”
“Rent due in 3 days.”
“Your subscription renewal hits tomorrow.”
Agentic behavior

It adapts to student rhythms:

Before lunch → budget cue
At grocery store → meal plan cue
Before weekend → spending forecast
After payday → savings allocation
Ridiculous version

A “sitsit financial damage predictor.”

Before a student party:

“Projected spending tonight: €47. Regret probability: 68%.”

Very Finnish student demo angle.

12. Banking + Health / Wellbeing
Stress-Aware Finance Agent

Financial stress is real. The wearable could avoid dumping stressful information at bad moments.

If user is in a meeting → suppress nonurgent alerts
If user is home and calm → show weekly summary
If suspicious fraud → interrupt anyway

This is a subtle but powerful agentic design point: timing matters.

Agentic behavior

The agent chooses when to communicate based on urgency and context.

Urgent + high risk → interrupt
Useful + nonurgent → wait
Sensitive + public place → hide details
Ridiculous version

A “financial therapist that refuses to show your balance in public.”

“You are in a crowded elevator. I will not traumatize you here.”

Funny, but privacy-aware.

13. Privacy and Consent Interfaces
Banking Privacy HUD

Because glasses are visible and ambient, the privacy problem is huge. But that can become the project’s strongest contribution.

The agent could use privacy modes:

Public mode:
Only vague cues, no amounts.

Private mode:
Detailed balances and transactions.

Secure mode:
Requires gesture + biometric/phone confirmation.

Panic mode:
Hide all financial information instantly.

Gesture examples:

Cover gesture = hide
Pinch-hold = unlock
Double tap = approve
Shake/flick = cancel
Agentic behavior

The agent classifies sensitivity.

Balance = sensitive
Fraud alert = urgent but must be discreet
Budget cue = low sensitivity
Transfer approval = high-risk, needs stronger confirmation
Ridiculous version

A “financial modesty filter.”

When someone else is nearby:

“Nosy person detected. Switching to vague vibes.”

Useful concept hidden inside a joke.

14. Social Banking
Shared Expense Agent

During dinner, travel, or events, the agent helps split costs.

“Group dinner total: €126.”
“Suggested split: €31.50 each.”
“Anna paid last time.”
“Gesture to send request.”
Agentic behavior

It understands social context and financial history.

Detects group payment
Suggests fair split
Drafts payment requests
Waits for approval
Learns recurring groups
Ridiculous version

A “friendship debt detector.”

At dinner:

“Reminder: Mikko still owes you €12 from March. Proceed with emotional caution.”

Comedic but relatable.

15. Shopping and Subscription Management
Subscription Ghost Hunter

When a recurring charge appears, the glasses show:

“Subscription renewed: €14.99.”
“You used this service 0 times last month.”
“Gesture to cancel later.”
Agentic behavior

The agent identifies waste and prepares next action.

Detects recurring charge
Checks usage or past behavior
Recommends cancel/keep/downgrade
Creates task or cancellation flow
Ridiculous version

A “subscription exorcist.”

“Another forgotten free trial has possessed your account.”

Great final presentation joke.

16. Investing and Savings
Goal-Based Savings Agent

Instead of showing abstract charts, the glasses show contextual progress.

“Trip to Japan: 64% funded.”
“This purchase delays goal by 2 days.”
“Round up this transaction?”

For investing, be careful. Regulated financial advice is sensitive. Better to frame it as education or goal tracking, not investment recommendations.

Agentic behavior
Tracks goal
Projects impact
Suggests safe non-advisory actions
Asks for confirmation
Ridiculous version

A “future self overlay.”

You look at a purchase and see:

“Future you would prefer the emergency fund.”

This could be emotionally powerful or unbearably annoying.

17. Banking for Visually Busy Contexts
Hands-Free Commuter Agent

During commuting, cycling, walking, or carrying bags, the user can check only what matters.

“Salary received.”
“Card charged unusually.”
“Upcoming bill tomorrow.”

This must be ultra-minimal.

One line
One icon
One gesture
No deep menus

The brief says the platform is glanceable and demands a different interaction model than a touchscreen, which is exactly the design constraint here. 

Nordea Wearable Banking Project

Ridiculous version

A “financial heads-up display for adulting.”

Bills incoming: 3
Safe-to-spend: €118
Impulse resistance: low
18. Insurance and Claims

Nordea may not want this if outside their exact banking scope, but financial services often overlap with insurance.

Claim Capture Agent

After an accident or lost item, the glasses guide the user through documentation.

“Take photo of damage.”
“Record location.”
“Ask for witness contact.”
“Upload receipt.”
“Draft claim.”
Agentic behavior

The agent knows the required evidence and guides the user in the moment.

Ridiculous version

A “bureaucracy sherpa.”

“You are emotionally distressed. I will now collect the boring details before you forget them.”

Actually useful.

19. Professional Banking / Wealth Advisor Mode
Advisor Meeting Overlay

For Nordea advisors, not consumers.

During a client meeting, the glasses could show:

Client goals
Risk profile reminder
Missing KYC information
Relevant product constraints
Follow-up tasks

But this is dangerous if it distracts from human connection.

Agentic behavior
Listens with consent
Surfaces relevant context
Flags compliance issues
Drafts meeting notes
Suggests next best question
Ridiculous version

A “compliance angel on the advisor’s shoulder.”

“You are about to imply guaranteed returns. Please do not.”

Very useful in regulated domains.

20. Banking in AR / Future Interfaces
Financial Layer on the World

This is the most futuristic direction.

The glasses do not show bank screens. They show financial meaning attached to situations.

At apartment viewing:
Affordability overlay

At store:
Budget impact overlay

At airport:
Currency and fee overlay

At restaurant:
Split bill overlay

At desk:
Financial planning overlay

The agent becomes a contextual interpreter of money in the world.

Ridiculous version

A full “money aura” system.

Everything has a financial glow:

Green = affordable
Yellow = think twice
Red = future pain
Purple = hidden subscription trap

Obviously too much, but visually excellent for a demo.

21. Anti-Use Cases: What Should NOT Be on Glasses

This project gets stronger if you explicitly say what does not belong there.

Bad wearable banking uses:

Full account management
Long forms
Detailed investment portfolio review
Complex loan applications
Reading terms and conditions
Entering sensitive personal data
Browsing transaction history for minutes
Anything visible to strangers

Better on phone/laptop:

Deep analysis
High-value transfers
Document signing
Dispute handling
Detailed advisory conversations

The glasses should be for:

Awareness
Alerts
Micro-decisions
Confirmations
Contextual nudges
Fast routing to phone/human support

That distinction will make your project look mature.

User is abroad and approaches an ATM/currency exchange.

Agent:

User pinches to compare options, taps to save travel note.

Context detected → financial risk interpreted → recommendation shown → user confirms.

Glanceable

Why it works:

My Honest Read

This project is less ridiculous than PALL0. It has a clearer reason to exist because banking has many moments where users need:

Fast awareness
Fraud protection
Contextual judgment
Private confirmation
Low-friction support

But it has one major trap:

Do not design “mobile banking, but worse and on glasses.”

The winning idea must be something where glasses are genuinely better than a phone.

The strongest principle:

Use glasses for the moment of financial awareness.
Use the phone or bank app for deep action.
Use the agent to decide what matters now.
Next Question or Task

The deepest question for this project is:

What financial moment is so time-sensitive, contextual, or hands-busy that a glanceable agent beats opening a banking app?

My strongest answers are: scam protection, travel money decisions, small business cashflow, and apartment affordability support.

Topic: What assets under management means in banking
Yes. Assets under management, or AuM, usually means the total value of customer money that a financial institution manages in investment or wealth products.
It commonly includes:
Mutual fundsETFsInvestment portfoliosPension/investment accountsDiscretionary wealth management portfoliosManaged savings productsOther long-term investment assets

So in the Nordea savings project, when they say Nordea wants to “grow AuM,” they likely mean:

Get more young customers to move money into Nordea-managed savings and investment products, not just keep cash in basic bank accounts.

Deposits = money customers hold at the bankAuM = investment/savings assets the bank manages or administers

Help young customers begin a trusted savings/investing relationship with Nordea, increasing long-term invested assets.
